Common Closure Principle (CCP) - Spring Boot 3.5

📌 Teorik Açıklama

Common Closure Principle (CCP), aynı nedenlerle değişecek olan sınıfların aynı pakette tutulması gerektiğini belirtir.

"Bir değişiklik birden fazla sınıfı etkiliyorsa, bu sınıflar aynı pakette gruplanmalıdır."

Bu ilke, özellikle bakım ve sürüm yönetimi açısından önemlidir. Bir iş kuralı değiştiğinde, ilgili tüm sınıfların tek bir paket altında olması, değişikliklerin daha kolay yönetilmesini sağlar.

Sonuç olarak: CCP uygulanmazsa aynı iş kuralına ait sınıflar farklı paketlere dağılır ve değişiklik yapmak karmaşık hale gelir. Uygulandığında ise değişiklikten etkilenen modüller minimize edilir.

🚀 Spring Boot 3.5 Örneği

❌ Yanlış Kullanım (CCP ihlali)

// Payment ile ilgili sınıflar farklı paketlerde dağılmış
com.example.user.UserPaymentValidator
com.example.order.OrderPaymentProcessor
com.example.logging.PaymentLogger

✅ Doğru Kullanım (CCP uygulanmış)

// payment paketi altında gruplanmış sınıflar
com.example.payment.PaymentValidator
com.example.payment.PaymentProcessor
com.example.payment.PaymentLogger

// örnek kullanım
package com.example.payment;

import org.springframework.stereotype.Service;

@Service
public class PaymentProcessor {
    private final PaymentValidator validator;

    public PaymentProcessor(PaymentValidator validator) {
        this.validator = validator;
    }

    public void processPayment(String user, double amount) {
        if (validator.validate(amount)) {
            System.out.println("Ödeme işlendi: " + user + " => " + amount);
        }
    }
}

Burada ödeme ile ilgili tüm sınıflar payment paketi altında toplanmıştır. Böylece ödeme kurallarında bir değişiklik yapılması gerektiğinde, sadece bu paket etkilenir.

🎯 Kazanımlar

  • Aynı iş kuralına bağlı sınıflar bir arada bulunur.
  • Değişikliklerin etkisi belirli paketle sınırlanır.
  • Bakım ve test süreçleri kolaylaşır.
  • Paketler daha anlamlı ve amaca uygun hale gelir.

💡 Kullanılması Durumunda Sağlayacağı Avantajlar

  • Yönetilebilirlik: Değişiklikler belirli paketlerle sınırlandığı için daha kolay yönetilir.
  • Bakım Kolaylığı: İlgili sınıflar aynı yerde olduğundan bakım süresi kısalır.
  • Sadelik: Paket yapısı daha anlamlı hale gelir.
  • Test Kolaylığı: Aynı işlevselliğe ait testler tek pakette toplanır.
  • Versiyonlama Avantajı: İlgili modüller aynı anda güncellenebilir.
