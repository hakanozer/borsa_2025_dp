Acyclic Dependencies Principle (ADP) - Spring Boot 3.5

📌 Teorik Açıklama

Acyclic Dependencies Principle (ADP), bağımlılıklar arasında döngüsel ilişkilerin olmaması gerektiğini belirtir.

"Paketler arası bağımlılık grafiği döngüsüz olmalıdır."

Yani, Package A → Package B → Package C → Package A gibi bir döngü olmamalıdır. Döngüler, bağımlılık cehennemi (dependency hell) yaratır ve değişiklikleri yönetilemez hale getirir.

Sonuç olarak: ADP uygulanmadığında, bir paketteki küçük bir değişiklik, dolaylı yoldan kendi üzerine geri döner. Uygulandığında ise bağımlılıklar yönlü ve yönetilebilir hale gelir.

🚀 Spring Boot 3.5 Örneği

❌ Yanlış Kullanım (ADP ihlali)

// Döngüsel bağımlılık
com.example.user → com.example.order → com.example.payment → com.example.user

Burada User paketi, Order'a bağımlı, Order ise Payment'a bağımlı, Payment ise tekrar User'a bağımlı hale gelmiş. Bu durum değişiklikleri yönetilemez hale getirir.

✅ Doğru Kullanım (ADP uygulanmış)

// Ortak bağımlılıkların soyutlanması
com.example.user
com.example.order
com.example.payment
com.example.shared (ortak bağımlılıklar burada)

// shared modülünde Interface tanımlanır
package com.example.shared;

public interface PaymentGateway {
    void process(double amount);
}

// payment modülü implement eder
package com.example.payment;

import com.example.shared.PaymentGateway;
import org.springframework.stereotype.Service;

@Service
public class PaymentService implements PaymentGateway {
    public void process(double amount) {
        System.out.println("Ödeme işlendi: " + amount);
    }
}

// order modülü sadece interface'e bağımlı olur
package com.example.order;

import com.example.shared.PaymentGateway;
import org.springframework.stereotype.Service;

@Service
public class OrderService {
    private final PaymentGateway paymentGateway;

    public OrderService(PaymentGateway paymentGateway) {
        this.paymentGateway = paymentGateway;
    }

    public void placeOrder(double amount) {
        paymentGateway.process(amount);
    }
}

Burada ortak bağımlılıklar shared paketine taşınarak bağımlılık döngüsü kırılmıştır.

🎯 Kazanımlar

  • Döngüsel bağımlılıklar ortadan kaldırılır.
  • Bağımlılık grafiği yönlü ve anlaşılır olur.
  • Değişiklikler daha kolay yönetilir.
  • Kod tabanı daha esnek ve ölçeklenebilir hale gelir.

💡 Kullanılması Durumunda Sağlayacağı Avantajlar

  • Bağımlılık Yönetimi: Döngüsüz bağımlılıklar, modüllerin birbirinden bağımsız gelişmesini sağlar.
  • Bakım Kolaylığı: Değişiklikler sınırlı etki alanına sahip olur.
  • Test Kolaylığı: Modüller bağımsız test edilebilir.
  • Modülerlik: Paketler arasındaki sınırlar daha net belirlenir.
  • Ölçeklenebilirlik: Yeni modüller eklenirken döngü riski minimize edilir.
