Stable Dependencies Principle (SDP) - Spring Boot 3.5

📌 Teorik Açıklama

Stable Dependencies Principle (SDP), "Daha az stabil olan paketler, daha stabil olanlara bağımlı olmalıdır." der.

Stabilite, bir paketin ne kadar değişmeyeceği ile ilgilidir. Eğer bir paket birçok paket tarafından kullanılıyorsa, o paket stabil kabul edilir ve değişmesi zor olmalıdır.

Sonuç olarak: SDP uygulanmadığında, kararsız bir pakete bağımlılık, tüm sistemi kırılgan hale getirir. Uygulandığında ise bağımlılıklar daha dayanıklı bir yapıya kavuşur.

🚀 Spring Boot 3.5 Örneği

❌ Yanlış Kullanım (SDP ihlali)

// notification paketi birçok pakete bağımlı
com.example.notification → com.example.user
com.example.notification → com.example.order

Burada notification modülü kararsızdır çünkü sürekli değişebilecek user ve order modüllerine bağımlıdır.

✅ Doğru Kullanım (SDP uygulanmış)

// shared paketi stabil bağımlılık içeriyor
com.example.shared (EmailService, SmsService interfaces)
com.example.notification (shared kullanır)
com.example.user (notification'a değil shared'e bağımlıdır)

// shared modülünde interface
package com.example.shared;

public interface MessageService {
    void sendMessage(String to, String message);
}

// notification modülü implement eder
package com.example.notification;

import com.example.shared.MessageService;
import org.springframework.stereotype.Service;

@Service
public class EmailService implements MessageService {
    @Override
    public void sendMessage(String to, String message) {
        System.out.println("Email gönderildi: " + to + " → " + message);
    }
}

// user modülü stabil shared'e bağımlı olur
package com.example.user;

import com.example.shared.MessageService;
import org.springframework.stereotype.Service;

@Service
public class UserService {
    private final MessageService messageService;

    public UserService(MessageService messageService) {
        this.messageService = messageService;
    }

    public void registerUser(String email) {
        System.out.println("Kullanıcı kaydedildi: " + email);
        messageService.sendMessage(email, "Hoşgeldiniz!");
    }
}

Burada kararsız bağımlılık zinciri kırılmış ve shared paketi stabil bağımlılık noktası olarak kullanılmıştır.

🎯 Kazanımlar

  • Kararsız bağımlılıklar ortadan kaldırılır.
  • Daha stabil paketler sistemin merkezine konumlanır.
  • Değişikliklerin etkisi minimize edilir.
  • Kod tabanı daha dayanıklı hale gelir.

💡 Kullanılması Durumunda Sağlayacağı Avantajlar

  • Dayanıklılık: Değişikliklerin en az hasarla yönetilmesini sağlar.
  • Bakım Kolaylığı: Stabil paketler, sistemin temel taşları olur.
  • Test Edilebilirlik: Stabil bağımlılıklar sayesinde sistem daha kolay test edilir.
  • Ölçeklenebilirlik: Yeni modüller stabil yapıya rahatlıkla entegre edilebilir.
  • Güvenilirlik: Sistemin çökme riskini azaltır.
