Stable Abstractions Principle (SAP) - Spring Boot 3.5

📌 Teorik Açıklama

Stable Abstractions Principle (SAP), "Bir paket ne kadar stabilse, o kadar soyut olmalıdır" der.

Yani stabil paketler (çok kullanılan, değişmesi zor olanlar) doğrudan somut sınıflar barındırmamalıdır. Bunun yerine soyutlamalar (interface, abstract class) içermelidir.

Sonuç olarak: Stabil ve soyut olan paketler değişime karşı dirençli olurken, kararsız paketler somut implementasyonları barındırarak esnekliği korur.

🚀 Spring Boot 3.5 Örneği

❌ Yanlış Kullanım (SAP ihlali)

// core modülünde somut sınıf
package com.example.core;

public class FileLogger {
    public void log(String message) {
        System.out.println("[LOG] " + message);
    }
}

// user modülü core'a bağımlı
package com.example.user;

import com.example.core.FileLogger;

public class UserService {
    private final FileLogger logger = new FileLogger();

    public void register(String username) {
        logger.log("Kullanıcı kaydedildi: " + username);
    }
}

Burada core modülü stabil olmasına rağmen soyutlama yerine somut sınıf içeriyor.

✅ Doğru Kullanım (SAP uygulanmış)

// core modülünde soyutlama
package com.example.core;

public interface Logger {
    void log(String message);
}

// infrastructure modülü implementasyon içerir
package com.example.infrastructure;

import com.example.core.Logger;
import org.springframework.stereotype.Service;

@Service
public class FileLogger implements Logger {
    @Override
    public void log(String message) {
        System.out.println("[LOG] " + message);
    }
}

// user modülü sadece soyutlamaya bağımlıdır
package com.example.user;

import com.example.core.Logger;
import org.springframework.stereotype.Service;

@Service
public class UserService {
    private final Logger logger;

    public UserService(Logger logger) {
        this.logger = logger;
    }

    public void register(String username) {
        logger.log("Kullanıcı kaydedildi: " + username);
    }
}

Burada core modülü sadece soyutlama içeriyor, somut implementasyonlar infrastructure gibi daha kararsız paketlerde bulunuyor.

🎯 Kazanımlar

  • Stabil paketler değişimden korunur.
  • Sistem daha modüler ve esnek hale gelir.
  • Bağımlılıkların yönü doğru şekilde tanımlanır.
  • Farklı implementasyonlar kolayca eklenebilir.

💡 Kullanılması Durumunda Sağlayacağı Avantajlar

  • Esneklik: Yeni implementasyonlar kolayca eklenebilir.
  • Dayanıklılık: Stabil paketler soyutlamalar sayesinde değişmeden kalabilir.
  • Test Kolaylığı: Mock veya stub ile test etmek kolaylaşır.
  • Genişletilebilirlik: Yeni ihtiyaçlara hızlı uyum sağlanır.
  • Temiz Mimari: Katmanlar arası bağımlılıklar daha düzenli hale gelir.
