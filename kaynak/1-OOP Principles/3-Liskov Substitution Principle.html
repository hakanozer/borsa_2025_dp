Liskov Substitution Principle (LSP) - Spring Boot 3.5

📌 Teorik Açıklama

Liskov Substitution Principle, SOLID prensiplerinin üçüncüsüdür ve türetilmiş sınıfların, türedikleri sınıfların yerine sorunsuzca geçebilmesi gerektiğini söyler.

Başka bir deyişle: "Bir alt sınıf, üst sınıfın tüm davranışlarını bozmayacak şekilde kullanılabilmelidir."

Eğer bir alt sınıf üst sınıfın kontratını bozarsa, bu durumda LSP ihlali söz konusu olur. Bu da polymorphism avantajlarını kaybettirir ve beklenmedik runtime hatalarına yol açar.

Sonuç olarak: LSP uygulanmazsa, inheritance (kalıtım) güvenilmez hale gelir ve sistem kırılganlaşır. Uygulanırsa ise sağlam, tahmin edilebilir ve genişletilebilir bir mimari elde edilir.

🚀 Spring Boot 3.5 Örneği

❌ Yanlış Kullanım (LSP ihlali)

package com.example.demo.model;

public class Bird {
    public void fly() {
        System.out.println("Kuş uçuyor...");
    }
}

public class Ostrich extends Bird {
    @Override
    public void fly() {
        // Devekuşu uçamaz! Bu, LSP ihlalidir.
        throw new UnsupportedOperationException("Devekuşu uçamaz!");
    }
}

✅ Doğru Kullanım (LSP uygulanmış)

package com.example.demo.model;

public interface Bird {
    void eat();
}

public interface FlyableBird extends Bird {
    void fly();
}

public class Sparrow implements FlyableBird {
    public void eat() {
        System.out.println("Serçe yemek yiyor.");
    }
    public void fly() {
        System.out.println("Serçe uçuyor.");
    }
}

public class Ostrich implements Bird {
    public void eat() {
        System.out.println("Devekuşu yemek yiyor.");
    }
    // Uçamaz, dolayısıyla fly metodu yok!
}

Burada Ostrich sınıfı Bird arayüzünü, Sparrow ise FlyableBird arayüzünü uygulamaktadır. Böylece devekuşu için uçma davranışı dayatılmamış olur. Bu, LSP’ye uygun bir çözümdür.

🎯 Kazanımlar

  • Polymorphism güvenilir şekilde kullanılabilir.
  • Kodun davranışı öngörülebilir hale gelir.
  • Alt sınıflar, üst sınıfların tüm beklentilerini karşılar.
  • Inheritance hiyerarşisi daha anlamlı ve doğru kurulur.

💡 Kullanılması Durumunda Sağlayacağı Avantajlar

  • Güvenilir Kalıtım: Tüm alt sınıflar üst sınıfın yerine geçebilir, runtime hataları engellenir.
  • Esnek Mimari: Yeni sınıflar eklenirken mevcut davranışlar bozulmaz.
  • Test Kolaylığı: Mock nesneler ve polimorfik testler daha kolay yazılır.
  • Yeniden Kullanılabilirlik: Kod tekrar kullanılabilir hale gelir.
  • Bakım Kolaylığı: Kod büyüdükçe kalıtım yapısı karmaşıklaşmaz.
