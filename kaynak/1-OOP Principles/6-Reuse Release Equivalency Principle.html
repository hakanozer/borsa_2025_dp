Reuse/Release Equivalency Principle (REP) - Spring Boot 3.5

📌 Teorik Açıklama

Reuse/Release Equivalency Principle (REP), bir yazılım bileşeninin tekrar kullanılabilir olması için aynı zamanda resmi bir sürüm yönetimine tabi tutulması gerektiğini belirtir.

Başka bir deyişle: "Bir bileşen yeniden kullanılabilir hale gelmeden, serbest bırakılabilir (release) bir sürüme sahip olmalıdır."

Bu ilke, özellikle kütüphane, modül ve paket geliştirme süreçlerinde önemlidir. Spring Boot projelerinde sık sık tekrar kullanılabilir modüller (örneğin starter bağımlılıkları) oluşturulur ve bunların her biri versiyonlanır.

Sonuç olarak: Eğer tekrar kullanılabilir kodlar sürüm yönetimine dahil edilmezse, farklı projelerde kullanılmaları zorlaşır, uyumsuzluklar artar ve bakım maliyeti yükselir. Sürüm yönetimi yapıldığında ise kod güvenilir ve sürdürülebilir olur.

🚀 Spring Boot 3.5 Örneği

✅ Doğru Kullanım (REP uygulanmış)

// common-logging kütüphanesi (ayrı bir module)
package com.example.common;

import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class LogHelper {
    private static final Logger logger = LoggerFactory.getLogger(LogHelper.class);

    public static void info(String message) {
        logger.info(message);
    }
}

// user-service module (common-logging'e bağımlı)
package com.example.userservice;

import com.example.common.LogHelper;
import org.springframework.stereotype.Service;

@Service
public class UserService {
    public void createUser(String name) {
        // iş mantığı
        LogHelper.info("Yeni kullanıcı oluşturuldu: " + name);
    }
}

Burada common-logging ayrı bir bağımsız modül olarak geliştirilmiş ve versiyon numarası verilerek başka servislere eklenmiştir (user-service gibi). Bu sayede aynı logging kütüphanesi farklı projelerde de kullanılabilir hale gelir.

🎯 Kazanımlar

  • Tekrar kullanılabilir bileşenler güvenilir bir sürüm yönetimi ile desteklenir.
  • Projeler arası uyumluluk artar.
  • Kütüphaneler bağımsız geliştirilip paylaşılabilir.
  • Bakım kolaylaşır, güncellemeler kontrollü yapılır.

💡 Kullanılması Durumunda Sağlayacağı Avantajlar

  • Stabilite: Farklı projelerde aynı versiyon kütüphane kullanılarak uyumsuzluk engellenir.
  • Paylaşılabilirlik: Modüller başka ekipler veya projelerle güvenle paylaşılır.
  • Tekrar Kullanım: Aynı modül farklı projelerde yeniden yazılmadan kullanılabilir.
  • Versiyon Kontrolü: Hangi sürümün nerede kullanıldığı net şekilde takip edilir.
  • Bakım Kolaylığı: Yeni sürüme geçiş süreçleri kolaylaşır.
